# External Integrations

**Analysis Date:** 2026-03-02

## APIs & External Services

**None Detected:**
- This is a standalone statistical pipeline with no external API calls
- No cloud service integrations, webhook handlers, or remote API clients
- All computation is local (R via pymer4, data processing via pandas/scipy)

## Data Storage

**Input Files:**
- **Excel (.xlsx)** - ABCD phenotype data (baseline or follow-up)
  - Client: `openpyxl`
  - Loaded via: `pandas.read_excel()` in `python_pipeline/preprocessing.py`
  - Column structure: 0-indexed continuous (cols 20–656) and binary (cols 670–1291) phenotypes
  - Example file: `data/FINAL_PHEWAS_baseline_n5556.xlsx`

- **CSV** - Alternative input for phenotype data or cluster labels
  - Client: `pandas.read_csv()`
  - Cluster labels CSV: subject_id_col, cluster_col (two columns)
  - Example file: `data/cluster_labels.csv`

**Output Files:**
- **CSV** - Results tables (one row per phenotype × cluster contrast)
  - Location: `{output_dir}/phewas_results_{suffix}.csv`
  - Written by: `python_pipeline/utils.py` (write_results function)
  - Columns: phenotype, cluster_contrast, beta, se, pval, converged, warning, domain, n_cases, etc.

- **PNG** - Visualization plots
  - Manhattan plot: `-log10(p)` by phenotype, colored by domain
  - Forest plot: Effect sizes (beta ± 95% CI) for significant phenotypes
  - Stacked bar: Domain breakdown of significant hits per cluster contrast
  - Generated by: `python_pipeline/visualizations.py` (matplotlib Agg backend)

**Configuration Files:**
- **YAML** - Domain configuration and example run config
  - `python_pipeline/configs/domains.yaml` - Phenotype → domain mappings via regex
  - `python_pipeline/configs/example_run.yaml` - Example configuration template

- **CSV** - Phenotype metadata
  - `python_pipeline/configs/phenotype_metadata.csv` - Authoritative variable → domain lookup
  - Loaded by: `python_pipeline/domains.py` for domain assignment

**Filesystem Only:**
- No external databases (SQLite, PostgreSQL, MongoDB, etc.)
- All data is file-based: .xlsx, .csv, .yaml
- No caching layer (in-memory pandas DataFrames)

## Monitoring & Observability

**Logging:**
- Python standard logging module
- Setup in: `python_pipeline/utils.py` (setup_logging function)
- Format: "YYYY-MM-DD HH:MM:SS | LEVEL | module | message"
- Levels: DEBUG, INFO, WARNING, ERROR
- Output: stderr (default Python logging behavior)
- No external logging service (Sentry, CloudWatch, etc.)

**Error Handling:**
- Try-catch blocks in model fitting (`python_pipeline/models.py`)
- Failed phenotypes return NaN results with `converged=False` and a `warning` string
- Errors logged but don't stop the parallel run; partial results are still valid
- Example failures: "insufficient_observations", "insufficient_binary_variation", "fit_failed"

## Environment Configuration

**Configuration Method:**
- YAML files (e.g., `python_pipeline/configs/example_run.yaml`)
- CLI flag overrides (take precedence over YAML)
- Dataclass validation in `python_pipeline/config.py` (PheWASConfig.validate)

**Required Arguments:**
- `--config` (YAML file path, required)
- `--timepoint` (baseline or followup, required for naming)
- `--phenotype-file` (Excel/CSV path, overrides config)
- `--cluster-file` (CSV with subject IDs and cluster labels, overrides config)
- `--output-dir` (where to write results, overrides config)

**Optional Arguments:**
- `--sex-stratum` (all, male, female)
- `--n-workers` (number of parallel processes)
- `--reference-cluster` (cluster label for intercept)
- `--checkpoint-file` (path to resumable checkpoint)
- `--plots-only` (re-generate plots from existing results CSV)

**Key Configuration Parameters:**
- `continuous_col_range` - 0-indexed columns for numeric phenotypes [20, 656]
- `binary_col_range` - 0-indexed columns for binary phenotypes [670, 1291]
- `skew_threshold` - |skewness| threshold for flagging columns (default 1.96)
- `winsorize_sd` - SD bounds for winsorizing (default 3.0)
- `optimizer` - GLMM optimizer name (default "bobyqa")
- `max_iterations` - Max optimizer iterations (default 100,000)
- `covariates` - List of covariate column names (default: C1–C10, sex, interview_age)

**Secrets/Credentials:**
- None (no .env file needed, no API keys)
- All paths are relative to CWD or absolute

## R Integration (via pymer4)

**External R Dependency:**
- R runtime (any recent version)
- R package: lme4 (for Lmer/Glmer GLMM fitting)
- Interface: rpy2 (Python ↔ R bridge via C)
- Location of calls: `python_pipeline/models.py`

**R Formula Execution:**
- Linear mixed-effects model (Lmer):
  ```
  phenotype ~ cluster_1 + cluster_2 + C1 + ... + C10 + sex + interview_age + (1|site_id) + (1|rel_family_id)
  ```
- Binomial GLMM (Glmer):
  ```
  phenotype ~ cluster_1 + cluster_2 + ... (same) ... family="binomial", nAGQ=0
  ```
- Optimizer: bobyqa (configurable)
- Control parameters: maxfun=100,000 (configurable)
