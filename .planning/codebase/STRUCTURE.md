## Directory Purposes

**Root Directory:**
- Purpose: Contains all pipeline scripts and data preparation code
- Contains: R scripts, shell scripts, RMarkdown document, README, input/output directories
- Key files: `multi_GWAS_CD2.sh`, `CD_sumstats.R`, `LDSC.R`, `PheWAS Analyses Resub5.Rmd`

**split_sumstats/**
- Purpose: Stores SNP chunk datasets for parallel processing
- Contains: Text files with summary statistics subsets
- Generated: Yes (created by `split_sumstats_CD2.R`)
- Committed: No (transient intermediate data)
- Subdirectories: split_sumstats2/ for main GWAS, split_sumstats3/ for independent factor models

**results/**
- Purpose: Accumulates GWAS results from 915 parallel job submissions
- Contains: CSV files with SNP-level association statistics per factor, SLURM logs
- Generated: Yes (created by multi_GWAS_CD2.sh and related job arrays)
- Committed: No (output data)
- Subdirectories: F1_sumstats/, F2_sumstats/, F3_sumstats/, F4_sumstats/ (one per latent factor)

**results_indep/**
- Purpose: Accumulates independent factor GWAS results (SNP effects constrained to single factor)
- Contains: CSV files with SNP-level results for factor-specific models
- Generated: Yes (created by independent factor job arrays)
- Committed: No (output data)
- Subdirectories: Compulsive/, Psychotic/, Neurodev/, Internal/ (one per factor)

**.planning/codebase/**
- Purpose: Codebase analysis documentation generated by GSD mapper
- Contains: ARCHITECTURE.md, STRUCTURE.md, and other codebase analysis documents
- Generated: Yes (created by automated analysis)
- Committed: Yes (reference documentation)

## Key File Locations

**Entry Points:**
- `cross_disorder_munge.sh`: Initial data munging pipeline entry (SLURM submission)
- `CD_sumstats.R`: Summary statistics preparation (standalone R execution)
- `LDSC.R`: Genetic covariance estimation (standalone R execution)
- `split_sumstats_CD2.R`: SNP chunk creation (standalone R execution)
- `multi_GWAS_CD2.sh`: Multivariate GWAS job submission (SLURM job array)
- `PheWAS Analyses Resub5.Rmd`: Final analysis and visualization (RMarkdown knit/render)

**Configuration:**
- No dedicated config files; parameters hardcoded in R scripts (sample sizes, trait names, file paths)
- SLURM directives embedded in shell scripts (--mem=4G, --cpus-per-task=8, --array=0-914)

**Core Logic:**
- `CD_sumstats.R`: sumstats() function from GenomicSEM for data harmonization
- `LDSC.R`: ldsc() function for covariance matrix computation
- `multi_GWAS_CD2.R`: userGWAS() function with SEM model specification; template for all GWAS analyses
- `multi_GWAS_ind_*.R`: Same userGWAS approach but with factor-constrained models

**Testing:**
- No formal test suite; validation via SLURM job logs and manual result inspection
- Output sanity checks: num_SNP_sets.txt records expected chunk count

## Naming Conventions

**Files:**
- R scripts: `[task]_[dataset/factor].R` (e.g., multi_GWAS_ind_Compulsive.R, CD_sumstats.R)
- Shell scripts: `[task]_[dataset/factor].sh` or standalone (e.g., cross_disorder_munge.sh, multi_GWAS_CD2.sh)
- Output: Factor results → `F[1-4]_sumstats/`, trait-specific → `results_indep/[Factor]/`
- Data: RData for serialized objects, CSV for tabular results, TXT for text summaries

**Directories:**
- `split_sumstats[N]/`: Numbered variants (split_sumstats2, split_sumstats3) for different pipeline runs/models
- `results/`: Aggregates all GWAS factor results
- `results_indep/`: Segregates independent factor model results by factor name

## Where to Add New Code

**New Analysis Stage (e.g., post-GWAS filter):**
- Primary code: Create new R script in root: `filter_results.R`
- Depends on: Results from `results/` or `results_indep/`
- SLURM wrapper: Create `filter_results.sh` if parallelization needed
- Output: New subdirectory in results/ or new directory (e.g., `filtered_results/`)
- Example: Following multi_GWAS_CD2.sh pattern

**New Factor Model Variant:**
- Template from: `multi_GWAS_ind_Compulsive.R`
- Modifications: Update factor model specification (lines 17-63), change output paths
- Execution: Create corresponding `.sh` SLURM wrapper copying `multi_GWAS_CD2_ind_Compulsive.sh` pattern
- Results directory: `results_indep/[NewFactor]/`

**New Phenotype Analysis:**
- Follow RMarkdown pattern in `PheWAS Analyses Resub5.Rmd`
- Create new .Rmd file in root (e.g., `Additional_PheWAS_Analyses.Rmd`)
- Load results from appropriate results/ subdirectory
- Dependencies: GenomicSEM, ggplot2, data.table, and visualization packages (see line 14-60 of Resub5)

**Utility Functions:**
- No shared utility library; helper functions defined inline in .Rmd file (see lines 69-76 of Resub5: withWarnings function)
- For reusable functions: Create new R file (e.g., `utils.R`) and source() in scripts needing it

## Special Directories

**split_sumstats/split_sumstats2/:**
- Purpose: Stores 915 SNP chunk files (6000 SNPs each) created by split_sumstats_CD2.R
- Generated: Yes
- Committed: No
- Lifecycle: Created once, read by all 915 multi_GWAS_CD2.sh job array tasks

**split_sumstats/split_sumstats3/:**
- Purpose: Alternative SNP chunk set used by independent factor models (multi_GWAS_ind_*.R scripts read from split_sumstats3/)
- Generated: Yes (likely output of alternative splitting run)
- Committed: No
- Lifecycle: Transient; generated separately from split_sumstats2 for variant analysis

**results/**
- Purpose: Central aggregation point for all factor-level GWAS results
- Generated: Yes (by multi_GWAS_CD2.sh job array, lines 69-81 of multi_GWAS_CD2.R write to ./results/F*/F*.csv)
- Committed: No
- Lifecycle: Accumulates across 915 parallel jobs; cleaned/analyzed downstream

**results_indep/**
- Purpose: Segregates independent factor GWAS results from correlated factor model results
- Generated: Yes (by independent factor job arrays)
- Committed: No
- Subdirectories per factor: Compulsive/, Psychotic/, Neurodev/, Internal/
